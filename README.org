
#TITLE: mnas-debug — инструменты для отладки

* Краткое описание

`mnas-debug` — небольшой вспомогательный набор рекомендаций, утилит и шаблонов рабочего процесса, предназначенный для упрощения отладки локальных Common Lisp-проектов.

Цель проекта:

- собрать в одном месте практики и команды для быстрой локальной диагностики проблем;
- показать надёжный рабочий процесс загрузки, повторной загрузки и тестирования систем через ASDF/Quicklisp;
- описать интеграцию отладки между взаимозависимыми проектами в `~/quicklisp/local-projects/`.

* Целевые сценарии использования

- Быстрая локальная отладка в REPL (SBCL, CCL, ECL и др.).
- Проверка регрессий при изменениях в коде через `asdf:test-system`.
- Изоляция минимального воспроизводимого примера для issue или PR.
- Использование в качестве вспомогательного набора для CI шагов (запуск тестов, сбор логов).

* Ключевые возможности

- Чёткие инструкции по быстрой загрузке и перезагрузке систем.
- Рекомендации по организации локальных зависимостей (Quicklisp local-projects / source-registry).
- Набор типовых команд и шаблонов рабочего процесса для воспроизведения и изоляции ошибок.

* Быстрый старт

- Разместите проект в `~/quicklisp/local-projects/` или настройте `*source-registry*` так, чтобы ASDF видел локальную копию.

- Пример загрузки в REPL:

  (ql:quickload :mnas-debug)

  ;; или, если предпочитаете напрямую через ASDF
  (asdf:load-system :mnas-debug)

* Примеры использования

- Загрузка mnas-debug и вашего проекта:

  (ql:quickload :mnas-debug)
  (ql:quickload :your-project)

- Повторная (гарантированная) загрузка при разработке:

  (asdf:operate 'asdf:load-op :your-project)

- Запуск тестов для обнаружения регрессий:

  (asdf:test-system :your-project)

* Рекомендованные практики отладки

- Изолируйте минимальный пример: скопируйте проблемный фрагмент в отдельный файл и загрузите его в голый REPL.
- Часто используйте `asdf:operate 'asdf:load-op` для последовательной загрузки и избежания частичных пересборок.
- Сохраняйте журнал (stdout/stderr) при воспроизведении ошибок — он часто содержит трассы и стек вызовов.
- Если проблема воспроизводится только в CI, повторите окружение локально (тот же Lisp-движок и версии зависимостей).

* Интеграция с другими проектами

1. Убедитесь, что отлаживаемый проект доступен ASDF (local-projects или source-registry).
2. В REPL загрузите `mnas-debug` и затем отлаживаемую систему, чтобы иметь доступ к общим советам и шаблонам.
3. При необходимости создавайте минимальные скрипты в `scripts/` (или аналогичной папке) для воспроизведения сценариев; храните их рядом с проектом.

* CI / автоматизация

- В CI добавляйте шаги для установки Quicklisp (или использования заранее настроенного окружения) и выполнения `asdf:test-system`.
- Артефакты, которые стоит собирать при падении тестов: лог-файлы, дампы (если применимо), вывод ошибок REPL.

* Полезные команды (сводка)

- `ql:quickload` — быстрая загрузка пакета через Quicklisp.
- `asdf:load-system` — загрузка локальной ASDF-системы.
- `asdf:operate 'asdf:load-op` — повторная полная загрузка системы.
- `asdf:test-system` — запуск тестов ASDF-системы.

* Что можно добавить дальше

- Примеры типичных паттернов логирования и минимальные сниппеты для включения в проекты.
